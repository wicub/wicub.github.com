改Chrome的User Agent,移动版网络

By Charles

2015-05-03 更新日期:2016-01-02

文章目录

 



理论上访问手机版或者iPad等平板电脑版的网络，应该可以剩些流量的，毕竟移动网络是经过优化压缩的，但是PC电脑如果访问移动版的网站呢？我主要使用的浏览器是Chrome，这几天也找了下Chrome下的修改方法，设置User Agent参数轻松把Chrome伪装成Android，iPhone，iPad等移动设备，访问针对移动版的网站。

 

修改方式比较常见的都是很长的一段参数，比如：

 

把Chrome伪装模拟成iPad, 修改Chrome的快捷方式，在快捷方式>目标一栏最后添加如下User Agent字符串：

 

–user-agent=”Mozilla/5.0(iPad; U; CPU iPhone OS 3_2 like Mac OS X;
en-us) AppleWebKit/531.21.10 (KHTML, like Gecko) Version/4.0.4
Mobile/7B314 Safari/531.21.10”

 

确定之后，重新打开Chrome访问网页(比如Google搜索，Gmail)，你会发现，网站变成了针对iPad优化过的界面。

 

上边的User Agent字符串是国际标准的格式，比较长，难记。今天就介绍个更简洁的方式修改Chrome的User Agent的方法,轻松体验移动版网络。

 

把Chrome伪装模拟成iPad的User Agent字符串：

 

–user-agent=”iPad”

 

把Chrome伪装模拟成iPhone的User Agent字符串：

 

–user-agent=”iPhone”

 

把Chrome伪装模拟成Android的User Agent字符串：

 

–user-agent=”Android”

 

这样是不是简洁方便很多，而且也容易记。

修改之后的Chrome快捷方式的目标(T)一栏应该为:

 

C:\Users\你的用户名\AppData\Local\Google\Chrome\Application\chrome.exe –user-agent=”iPad”

 

更新提示：好像横线在网页中显示效果不同，以上User Agent开头都是两个横线。

 

在Chrome下测试成功，Firefox和Opera等其他浏览器不晓得支持不支持这种简洁的User
Agent，我这里没装其他的浏览器，无法测试，看到本文的童鞋有时间的话可以测试下。按照你希望伪装模拟的移动设备设置对应的User
Agent，体验移动版网站。

 

使用这个Chrome插件Window Resizer轻松调整Chrome的窗口尺寸配合移动版页面。

 

ps：对于像我一样这样使用3g网络的朋友来说，主要在意的还是能够省些流量
